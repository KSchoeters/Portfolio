CropWatchers

Project Information
  • Type: Group Project
  • Tech stack: React, FastAPI, AWS, Terrafrom, Python, ChartJS, MongoDB
  • Duration: Sep 2024 – Feb 2025
  • Client: VITO
Documents
  • Concept Document (NL)
  • Promo Video
Description

CropWatchers is a web-based platform built with React, designed to assist farmers in reporting and managing crop damage. The application enables farmers to register their agricultural fields and submit detailed damage claims. Each claim can be supported with multiple types of evidence, including regional news articles, local weather conditions, and satellite imagery collected directly within the platform. This ensures that reports are well-documented and substantiated, increasing their credibility and usability for insurance and governmental review.

In addition to individual use by farmers, the system offers role-based access control. Consultants can be added to a farmer’s field, giving them permission to view field data and submitted claims, fostering collaboration and expert oversight. For governmental agencies, a dedicated dashboard provides insights into regional trends, showing the number of claims submitted, affected areas, and estimated damage costs. This data can help inform policy decisions, prioritize responses, and allocate financial resources effectively.

My Contributions

My role in the project involved several core features focused on automation, AI integration, and user guidance. I developed a contextual chatbot integrated into the application to assist users by answering common questions and guiding them through functionalities like field registration or damage report creation. This lowered the barrier for new users and improved overall user experience.

I also implemented a web scraping service that retrieves news articles related to weather and environmental incidents in the specific region and timeframe of a reported claim. This allows users to attach relevant third-party media as supporting evidence for their reports. To streamline official documentation, I built an automated PDF generation pipeline that compiles all claim data and attachments into a well-formatted, downloadable report.

Lastly, I integrated EOplaza geospatial services to analyze satellite imagery associated with each field. This integration enabled the system to automatically detect anomalies such as potential crop stress or damage and estimate yield levels. These insights are presented directly in the platform and serve as another layer of verification for submitted claims.

BI Project

Project Information
  • Type: Individual Project
  • Tech stack: Qlik, Replit
  • Duration: Sep 2024 – Jan 2025
  • Client: Epic Data
Documents
  • Functional Specifications (EN)
Description

This project centered around the creation of a comprehensive data analytics and visualization solution for Tomorrowland’s financial data using Qlik. The initial phase began with the analysis of several Excel files containing raw operational and financial information. From this broader dataset, the scope was narrowed to focus specifically on revenue generated through merchandise and beverage sales, two of the most significant commercial streams during the festival.

A star schema model was designed to organize the data efficiently for reporting and analysis. This involved identifying appropriate fact and dimension tables, cleaning and standardizing the data, and building relationships that would allow for robust querying and filtering. The scripting capabilities within Qlik were used to load and transform the data into this star model, laying the foundation for a responsive and scalable dashboard.

On top of the data model, an interactive Qlik dashboard was developed to visualize the most relevant financial insights. The dashboard included detailed charts and KPIs that highlighted overall revenue, category-wise sales performance and seasonal fluctuations. User interactions were enabled through filters and drill-down capabilities, allowing stakeholders to explore the data from multiple angles, such as time, product category, or sales channel.

To enhance communication of the findings beyond the dashboard interface, a complementary data journalism piece was created using Replit. This web-based report presented a narrative-driven summary of key insights, enriched with embedded visualizations from the Qlik dashboard. The goal was to translate raw data into a compelling and accessible story that could be shared with both technical and non-technical audiences.

Overall, the project delivered a full pipeline solution—from raw data ingestion and modeling, to visual storytelling and stakeholder engagement—demonstrating the practical use of business intelligence tools to extract value from real-world event data.

ECS Project

Project Information
  • Type: Group Project
  • Tech stack: PowerBI
  • Duration: Sep 2024 – Jan 2025
  • Client: Frixion
Documents
  • Demo Video
Description

This project focused on developing a centralized SharePoint-based application to support core administrative workflows. The platform provides structured views for incoming and outgoing invoices, a dedicated overview page for ongoing projects with individual project detail pages, and a company-wide phonebook containing essential employee contact and role information. The goal was to create a cohesive internal tool that improves transparency, document management, and operational efficiency across departments.

My Contributions

I was responsible for the automation of the invoice intake process within the SharePoint environment. Using Power Automate, I set up a workflow where emails sent to a specific inbox were automatically monitored. Whenever an email arrived with an invoice PDF as an attachment, the file was uploaded to a SharePoint document library. AI capabilities were then used to extract key data from the PDF—such as company name, amount, due date, and other fields—to generate a structured invoice entry.

To ensure timely follow-up, I configured weekly summary emails listing all newly processed invoices. I also built automated reminder flows that notify users of invoices nearing or past their due dates, helping maintain accurate and prompt financial handling.

In addition to the invoice automation, I developed a Power BI dashboard embedded directly into the SharePoint platform. This dashboard visualizes invoice data with charts and KPIs, offering insights into payment trends, outstanding balances, and vendor-specific statistics. To further enhance usability, I integrated a Power Apps component within the dashboard, allowing users to directly edit invoice metadata such as status or amount, streamlining small corrections without leaving the reporting view.

NLP Project

Project Information
  • Type: Individual Project
  • Tech stack: Python, CrewAI, Streamlit
  • Duration: Nov 2024 – Dec 2024
  • Client: School
Documents
  • NLP Report (EN)
  • GitHub Repository
Description

This project involved building an intelligent chatbot platform using CrewAI, powered by a LLaMA model hosted on Groq Cloud. The primary goal was to create a conversational assistant capable of answering user questions with up-to-date information gathered from the internet, while ensuring transparency by citing all sources used in its responses. The result is a chatbot that not only delivers accurate answers but also promotes user trust by providing clear references for every response.

The chatbot architecture is designed to be both responsive and adaptable. Whenever a question is asked, the system performs a real-time web search, curates relevant content, and processes it through the LLaMA model to generate a coherent and informative reply. Each response includes direct links to the original content, giving users the opportunity to verify the sources themselves or explore further.

In addition to internet-based querying, the chatbot also supports memory-based interaction through PDF ingestion. Users can upload documents directly to the platform, and the chatbot stores their contents in its memory. Once stored, the bot can reference those PDFs when answering questions—allowing for contextual replies grounded in private or domain-specific documents, such as reports, manuals, or research papers.

Together, these capabilities make the chatbot a powerful dual-purpose assistant: one that can dynamically search and validate external information, while also serving as a personalized knowledge tool for any content uploaded by the user.

Trackmania Self Driving Car

Project Information
  • Type: Group Project
  • Tech stack: Python
  • Duration: Sep 2024 – Nov 2024
  • Client: School
Documents
  • Trackmania Report (EN)
  • Demo Video
Description

This project focused on building a self-driving car system within the Trackmania environment. The goal was to develop an AI model capable of navigating the in-game track autonomously using computer vision and supervised learning techniques. The development process began with the collection of gameplay data by manually driving through tracks while capturing a large set of in-game screenshots alongside the corresponding key presses. These image-action pairs formed the foundation of the training dataset.

Using this dataset, we trained a neural network model to predict driving actions based on visual input. Once the model had been trained, it was integrated into the game for testing, allowing it to take control of the car and attempt autonomous navigation. After each testing phase, model performance was evaluated, and the insights gained were used to further tune the architecture, training parameters, and data preprocessing pipeline. This loop of retraining and testing was repeated iteratively to gradually improve the model’s driving accuracy and consistency.

My Contributions

My responsibilities centered around building and refining the machine learning model. I wrote the core model architecture, handled data preprocessing, and implemented training scripts using frameworks such as TensorFlow or PyTorch. A significant part of my contribution involved experimenting with different hyperparameters and training strategies to optimize model performance. I also ensured that all components were clearly documented.
